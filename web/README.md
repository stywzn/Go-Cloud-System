# Go Cloud System - 前端界面

## 🌟 功能特性

### 🎯 系统监控
- **实时状态检查**: 自动检测所有微服务健康状态
- **服务状态展示**: Gateway、Storage、Interaction服务在线状态
- **模拟在线用户**: 展示系统活跃度

### 🎁 抽奖系统
- **一键参与**: 点击按钮即可参与抽奖
- **实时反馈**: 显示抽奖结果和中奖信息
- **错误处理**: 友好的错误提示和状态反馈

### ☁️ 文件存储
- **拖拽上传**: 支持文件选择和上传
- **进度显示**: 实时上传进度条
- **秒传支持**: 基于文件Hash的去重上传

### 🔧 系统管理
- **架构信息**: 详细的系统架构和技术栈说明
- **性能指标**: QPS、延迟、并发支持等关键指标
- **核心特性**: 链路追踪、死信队列、优雅停机等功能展示

## 🚀 快速开始

### 1. 启动完整系统
```bash
# 启动所有中间件
make infra-up

# 启动所有微服务
make run-all

# 启动前端界面 (新终端)
make run-web
```

### 2. 访问界面
- **前端界面**: http://localhost:8084
- **API网关**: http://localhost:8080
- **RabbitMQ管理**: http://localhost:15672 (guest/guest)
- **MinIO控制台**: http://localhost:9001 (admin/password123)

## 🎮 使用指南

### 抽奖流程
1. 系统启动后，前端会自动获取认证Token
2. 点击"参与抽奖"按钮
3. 查看抽奖结果（成功获得4GB存储配额或失败提示）

### 文件上传
1. 点击"选择文件上传"
2. 选择任意文件进行上传
3. 查看上传进度
4. 上传成功后可在Storage服务中查看

### 系统监控
1. 页面顶部显示各服务实时状态
2. 每30秒自动更新状态信息
3. 点击"查看详情"了解系统架构

## 🔧 技术实现

### 前端技术
- **纯HTML/CSS/JavaScript**: 无框架依赖，轻量级实现
- **响应式设计**: 适配不同屏幕尺寸
- **实时通信**: 基于Fetch API的HTTP请求
- **用户体验**: 动画效果、进度条、通知提示

### 后端集成
- **Gateway认证**: 自动获取JWT Token
- **微服务通信**: 通过Gateway转发到各服务
- **错误处理**: 统一的错误处理和用户反馈
- **状态管理**: 实时服务状态检查

## 🔍 故障排除

### 常见问题
1. **服务状态检查失败**: 确保所有微服务已启动
2. **抽奖失败**: 检查RabbitMQ和Redis连接
3. **文件上传失败**: 确认MinIO服务正常运行
4. **Token获取失败**: 检查Gateway服务状态

### 调试技巧
1. 打开浏览器开发者工具查看网络请求
2. 检查各服务的日志输出
3. 使用curl命令直接测试API接口
